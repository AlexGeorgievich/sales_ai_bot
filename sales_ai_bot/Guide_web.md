# Руководство по развертыванию Sales AI Bot в облаке 🚀

Это подробное пошаговое руководство по бесплатному развертыванию проекта **Sales AI Bot** в сети. Оно составлено так, чтобы даже специалист начального уровня (Junior) смог запустить систему с нуля без ошибок.

---

## 🏗️ Архитектура продакшн-решения

В локальной разработке мы использовали связку из 5 сервисов (Nginx, FastAPI, Streamlit, PostgreSQL, Redis) через Docker Compose. В облаке для оптимизации ресурсов и использования **бесплатных тарифов** мы применили более простую схему:

1. **База данных PostgreSQL**: Размещается на облачном сервисе **Neon.tech** (бесплатно, навсегда).
2. **Кэш Redis**: Размещается на облачном сервисе **Upstash.com** (бесплатно, навсегда).
3. **Бэкенд + Лендинг (FastAPI)**: Размещается на **Render.com** (бесплатно, Web Service). Бэкенд самостоятельно раздает файлы сайта и виджета с корневого адреса `/`, исключая необходимость в Nginx.
4. **Админ-панель (Streamlit)**: Размещается на **Render.com** (бесплатно, второй Web Service).

## 🛠️ Подготовка: Облачные базы данных

Перед началом деплоя серверов нам понадобятся доступы к базе данных и кэшу.

### Шаг 1. Создание PostgreSQL на Neon.tech
1. Перейдите на сайт **[Neon.tech](https://neon.tech/)** и зарегистрируйтесь с помощью своего аккаунта GitHub.
2. Создайте новый проект (назовите его, например, `sales-bot-db`).
3. В дашборде проекта найдите блок **Connection Details**.
4. В выпадающем списке выберите **SQLAlchemy**. Скопируйте полученную строку. Она будет выглядеть так:
   `postgresql://alex:password@ep-cool-water-12345.us-east-2.aws.neon.tech/neondb?sslmode=require`

⚠️ **КРИТИЧЕСКИЙ НЬЮАНС (Совместимость с asyncpg)**:
1. Замените протокол в начале строки с `postgresql://` на **`postgresql+asyncpg://`** (это нужно FastAPI для асинхронного подключения).
2. **Обязательно удалите** из конца строки параметры `?sslmode=require` (или любые другие после знака `?`). Асинхронный драйвер Python (`asyncpg`) не поддерживает этот параметр в строке запроса и упадет при запуске с ошибкой `unexpected keyword argument 'sslmode'`.
* **Правильный вид строки**: `postgresql+asyncpg://alex:password@ep-cool-water-12345.us-east-2.aws.neon.tech/neondb` (Сохраните её как `DATABASE_URL`).

---

### Шаг 2. Создание Redis на Upstash.com
1. Перейдите на сайт **[Upstash.com](https://upstash.com/)** и войдите через GitHub.
2. Нажмите кнопку **Create Database**.
3. Выберите тип базы **Redis**, укажите название (например, `sales-bot-cache`) и выберите ближайший регион.
4. В настройках базы найдите строку подключения **Redis Connect URL** (вкладка `Web / API` -> `iOREDIS`). Она выглядит так:
   `redis://default:password@ep-cool-redis-12345.upstash.io:6379/0`

⚠️ **КРИТИЧЕСКИЙ НЬЮАНС (Шифрование SSL)**:
Облачный Redis от Upstash требует защищенного SSL-соединения. Если подключиться по обычному `redis://`, сервер мгновенно сбросит соединение с ошибкой `Connection closed by server`.
* Замените протокол в начале строки на **`rediss://`** (с двумя буквами **`s`** на конце).
* **Правильный вид строки**: `rediss://default:password@ep-cool-redis-12345.upstash.io:6379/0` (Сохраните её как `REDIS_URL`).

## 🚀 Деплой на Render.com

Для деплоя ваш код должен быть запушен в ваш репозиторий на GitHub.

### Шаг 3. Деплой Бэкенда и Лендинга (FastAPI)
1. Авторизуйтесь на **[dashboard.render.com](https://dashboard.render.com/)** через GitHub.
2. В верхнем меню нажмите кнопку **New +** и выберите **Web Service**.
3. Подключите ваш репозиторий `sales_ai_bot`.
4. Заполните основные настройки:
   * **Name**: `sales-bot-backend`
   * **Runtime**: Выберите **Docker** (Render соберет проект по нашему `Dockerfile`).
   * **Instance Type**: Выберите **Free** ($0/mo).
5. ⚠️ **КРИТИЧЕСКИЙ НЬЮАНС (Папка проекта)**:
   Наш репозиторий имеет вложенную структуру: папка `sales_ai_bot` находится внутри корня. Если оставить настройки по умолчанию, сборка упадет с ошибкой `failed to read dockerfile: no such file or directory`.
   * Найдите поле **Root Directory** и впишите туда: `sales_ai_bot`.
6. Оставьте поле **Docker Command** пустым (проект автоматически применит команду запуска Uvicorn из Dockerfile).
7. Нажмите кнопку **Advanced** в самом низу страницы настроек и найдите раздел **Environment Variables** (Переменные окружения). Нажмите **Add Environment Variable** и добавьте следующие ключи:
   * `DATABASE_URL` = *(ваша строка asyncpg из Шага 1)*
   * `REDIS_URL` = *(ваша строка rediss из Шага 2)*
   * `GIGACHAT_API_KEY` = *(ваш ключ API GigaChat)*
   * `TELEGRAM_BOT_TOKEN` = *(токен вашего Telegram-бота)*
   * `TELEGRAM_CHAT_ID` = *(ваш личный Telegram ID)*
   * `PROJECT_NAME` = `Sales AI Bot`
   * `DEBUG` = `false`
8. Нажмите кнопку **Create Web Service** в самом низу.
   * *Render начнет сборку Docker-образа. Это займет 3–5 минут. После успешной сборки статус сменится на **Live**, а вверху страницы появится ваша публичная ссылка, например: `https://sales-bot-backend-xxxx.onrender.com`.*

---

### Шаг 4. Деплой Админ-панели (Streamlit)
⚠️ **ВАЖНО**: Нельзя изменять команду запуска (Docker Command) в первом сервисе бэкенда! Для админ-панели мы должны создать **второй, отдельный** Web Service на Render.

1. В дашборде Render снова нажмите **New +** -> **Web Service**.
2. Выберите тот же самый репозиторий `sales_ai_bot`.
3. Заполните настройки:
   * **Name**: `sales-bot-admin`
   * **Runtime**: **Docker**
   * **Instance Type**: **Free**
   * **Root Directory**: `sales_ai_bot` *(обязательно, как на Шаге 3)*
4. ⚠️ **КРИТИЧЕСКИЙ НЬЮАНС (Команда запуска Streamlit)**:
   По умолчанию Dockerfile запускает бэкенд FastAPI. Нам нужно принудительно переопределить эту команду для запуска интерфейса Streamlit.
   * Найдите поле **Docker Command** (оно находится под Root Directory) и впишите туда команду:
     `streamlit run admin.py --server.port 10000 --server.address 0.0.0.0`
5. Нажмите кнопку **Advanced** внизу страницы и перейдите в **Environment Variables**. Добавьте переменные:
   * `DATABASE_URL` = *(из Шага 1)*
   * `REDIS_URL` = *(из Шага 2)*
   * `PROJECT_NAME` = `Sales AI Bot`
   * `ADMIN_USERNAME` = *(придумайте логин для входа в админку, например `manager`)*
   * `ADMIN_PASSWORD` = *(придумайте надежный пароль для входа)*
6. Нажмите **Create Web Service**.
   * *После завершения сборки статус сменится на **Live**, и вы получите вторую уникальную ссылку для админ-панели, например: `https://sales-bot-admin-xxxx.onrender.com`.*

## 🩺 Диагностика проблем и их решение

Если в процессе деплоя что-то пошло не так:

| Проблема | Симптом / Лог ошибки | Решение |
| :--- | :--- | :--- |
| **Ошибка сборки Docker** | `failed to read dockerfile: no such file or directory` | Проверьте поле **Root Directory** в разделе **Settings** на Render. Там должно быть написано `sales_ai_bot`, а не просто `/`. |
| **Падение на старте (Бэкенд)** | `TypeError: connect() got unexpected keyword argument 'sslmode'` | Вы забыли удалить `?sslmode=require` в конце переменной `DATABASE_URL`. Перейдите во вкладку **Environment**, отредактируйте переменную и сохраните изменения. |
| **Падение на старте (Redis)** | `Connection closed by server` в логах | Проверьте префикс переменной `REDIS_URL`. Он должен быть строго `rediss://` (с двумя **s**), а не `redis://`. |
| **Ошибка 403 в Telegram** | `Forbidden: the bot can't send messages to the bot` | В переменной `TELEGRAM_CHAT_ID` указан ID самого бота (или неверный ID). Напишите в Telegram боту `@userinfobot`, получите ваш личный числовой ID и вставьте его в настройки. Не забудьте нажать `/start` в чате с вашим ботом. |
| **Обновление настроек не работает** | Переменные изменены, но ошибка осталась | При изменении файла `.env` на компьютере Docker-контейнеры в облаке не обновляются сами. Нажмите кнопку **Manual Deploy** -> **Clear Cache and Deploy** на Render для применения изменений. |

## 📈 Проверка работоспособности системы

1. Откройте ссылку вашего бэкенда (`https://sales-bot-backend-xxxx.onrender.com`).
2. Перед вами появится лендинг IT-компании. Откройте чат-виджет в правом нижнем углу и напишите сообщение боту.
3. Откройте ссылку вашей админки (`https://sales-bot-admin-xxxx.onrender.com`).
4. Введите логин и пароль, которые вы указали в переменных окружения админки.
5. Перейдите во вкладку **История диалогов** — вы увидите вашу переписку с ботом.
6. Заполните форму заказа на сайте (например, B2B-форму). Лид моментально отобразится во вкладке **Лиды** в админке, а в ваш Telegram придет красивое пуш-уведомление!
